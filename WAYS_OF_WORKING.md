# Ways of Working

This document outlines the collaborative workflows for the SAFe agents in this project.

## Epic to Feature to Story

This workflow describes how an Epic is broken down into Features and then into Stories.

1.  **Epic Owner:** Creates the Epic, including the Epic Hypothesis Statement and the business case.
2.  **Portfolio Management:** Reviews and approves the Epic.
3.  **Epic Owner, Product Management, and System Architect:** Collaborate to decompose the Epic into Features.
4.  **Product Management:** Prioritizes the Features in the Program Backlog using WSJF.
5.  **Product Owner:** Pulls Features from the Program Backlog and works with the Agile Team to decompose them into Stories.

## PI Planning

This workflow describes how the agents collaborate during PI Planning.

1.  **RTE:** Facilitates the PI Planning event.
2.  **Product Management:** Presents the Product Vision and the top 10 Features.
3.  **System Architect:** Presents the Architectural Vision.
4.  **Agile Teams (with Product Owner and Scrum Master):**
    *   Break down Features into Stories.
    *   Estimate the Stories.
    *   Create their PI Objectives.
    *   Identify dependencies and add them to the Program Board.
5.  **RTE and Scrum Masters:** Facilitate the Scrum of Scrums to manage dependencies.
6.  **Business Owners (represented by Product Management):** Approve the PI Objectives.

## Dependency Management

This workflow describes how dependencies between teams are managed.

1.  **Agile Teams:** Identify dependencies during PI Planning and add them to the Program Board.
2.  **Scrum Masters:** Use the Program Board to track dependencies and facilitate conversations between teams.
3.  **RTE:** Manages program-level dependencies and escalates impediments as needed.

## Metrics and Reporting

This workflow describes how metrics are collected and reported.

1.  **Agile Teams:** Track their Iteration Velocity.
2.  **RTE:**
    *   Aggregates team velocities to calculate the Program Velocity.
    *   Calculates the Program Predictability Measure.
    *   Tracks Flow Metrics (Cycle Time, Throughput, WIP).
3.  **RTE and Product Management:** Present the metrics during the Inspect & Adapt workshop.
