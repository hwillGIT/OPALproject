# open.epic :: Data Sharing Playbooks

> Source: https://open.epic.com/Playbooks

---

# Data Sharing Playbooks

 
Data Sharing Playbooks offer practical recommendations for connecting with Epic community members. Each playbook highlights proven approaches to common data sharing use cases, whether supporting patient-directed access, enabling providers to deliver whole-person care, or improving operations. By leveraging open standards, these playbooks help innovators accelerate safe, reliable connections.

 

 

 
 

 
 

 

 

 [
 
## Alert Managers

 
Alert Managers ingest alerts from a variety of devices, such as inpatient vitals monitors, and determine who should get notified for each alert via Epic's Alert Communicator. 

 ](https://fhir.epic.com/Documentation?docId=alert_managers_playbook)
 

 

 [
 
## Ambulatory Cardiac Devices

 
Wearable cardiac devices like Holter monitors, event monitors, long-term continuous monitors and mobile cardiac telemetry, are used to monitor and record cardiac data while a patient is away from the office or hospital.

 ](https://fhir.epic.com/Documentation?docId=wearable_cardiac_devices_playbook)
 

 

 [
 
## Automated Dispensing System

 
Automated Dispensing Systems (ADS) are computerized devices that store, dispense, and track medications at the point of service by receiving patient and order information to determine available medications and send dispense information back to Epic when medications are removed.

 ](https://fhir.epic.com/Documentation?docId=dispensing_playbook)
 

 

 [
 
## Bayesian Medication Dosing Decision Support

 
A decision support system launching from Hyperspace to pull patient and medication data and provide personalized, Bayesian-based dosing recommendations for medications.

 ](https://fhir.epic.com/Documentation?docId=bayesianmedicationdosing_playbook)
 

 

 [
 
## Bedside Dining

 
Bedside Dining applications present patients dining menus and options to place dining orders within MyChart Bedside.

 ](https://fhir.epic.com/Documentation?docId=bedsidedining_playbook)
 

 

 [
 
## Clinical Decision Support

 
Trigger real-time, end user-facing decision support with CDS Hooks.

 ](https://fhir.epic.com/Documentation?docId=cds_playbook)
 

 

 [
 
## Clinical Knowledge Set Apps

 
The Clinical Knowledge Set integration is based on the HL7 Infobutton standard, which allows clinicians to retrieve targeted information provided by third parties, specific to the context of the patient or clinical workflow.

 ](https://fhir.epic.com/Documentation?docId=clinical_knowledge_set_playbook)
 

 

 [
 
## Computer Telephony Integration (CTI)

 
Contact center agents and other Epic users can make and receive phone calls directly from Hyperspace with computer telephony integration (CTI).

 ](https://fhir.epic.com/Documentation?docId=cti_playbook)
 

 

 [
 
## Deploying Predictive Models

 
Deploy a predictive model on a health system's Epic instance using Epic's Cognitive Computing Platform or other APIs.

 ](https://fhir.epic.com/Documentation?docId=predictive_models_playbook)
 

 

 [
 
## Digital Pathology

 
Digital Pathology is an integration between Beaker (Epic's laboratory application) and a 3rd party Image Management System (IMS) to allow pathologists to gain access via a hyperlink to high-resolution, digitized images of case slides for examination and interpretation. 

 ](https://fhir.epic.com/Documentation?docId=digital_pathology_playbook)
 

 

 [
 
## ECG Management System

 
Electrocardiogram (ECG) management systems store and interpret waveforms produced from an ECG machine, associate the waveform with an order for a patient, and file the data back to Epic.

 ](https://fhir.epic.com/Documentation?docId=ecgmanagementsystem_playbook)
 

 

 [
 
## ED Pre-Arrival EMS Workflows

 
Support EMS to ED pre arrival coordination, identity matching, and timely filing of prehospital data to the patient chart.

 ](https://fhir.epic.com/Documentation?docId=ed_pre_arrival_ems_playbook)
 

 

 [
 
## Encoder

 
Encoder integrations use a third-party system to determine and file diagnosis, procedure, and DRG codes based on clinical documentation

 ](https://fhir.epic.com/Documentation?docId=encoderplaybook)
 

 

 [
 
## Endoscope Data Capture

 
Endoscope Data Capture integrations allow vendors to gather data from a scope, format it, and then send it downstream to be used throughout Lumens workflows, e.g., documenting findings or writing a procedure note.

 ](https://fhir.epic.com/Documentation?docId=endoscope_data_capture_playbook)
 

 

 [
 
## End-User Authentication

 
End-User Authentication integrations may include support for Login or Reauthentication workflows for users accessing Hyperdrive, Haiku, Canto, or Rover.

 ](https://fhir.epic.com/Documentation?docId=enduserauthentication_playbook)
 

 

 [
 
## External Health Records for Patient Treatment

 
Epic provides seamless interoperability solutions to ensure healthcare providers can access the right patient information at the right time, using widely-adopted standards and tools for efficient data exchange across networks such as TEFCA and Carequality.

 ](https://fhir.epic.com/Documentation?docId=external_health_records_for_patient_treatment)
 

 

 [
 
## Fully Autonomous Coding

 
Fully Autonomous Coding applications use AI and machine learning to receive and process patient information, clinical documentation, results, and charges to then send updated charges back for filing to Professional or Hospital Billing.

 ](https://fhir.epic.com/Documentation?docId=autonomouscoding_playbook)
 

 

 [
 
## Hemodyamics System

 
Hemodynamic Monitoring Systems are used in Cardiac Cath Labs, Cardiac EP Labs, and some Interventional Radiology Labs to monitor patients' cardiac vitals and pressures during invasive procedures.

 ](https://fhir.epic.com/Documentation?docId=hemodynamics_system_playbook)
 

 

 [
 
## Home Dialysis

 
Home dialysis machines are used by patients to administer hemodialysis and peritoneal dialysis at home.

 ](https://fhir.epic.com/Documentation?docId=rpmtutorial)
 

 

 [
 
## Implantable Cardiac Devices

 
The IHE Implantable Device – Cardiac Observations (IDCO) profile is used to communicate data from an implantable cardiac device (such as a pacemaker or ICD) with Epic and allows that data to be incorporated into the patient chart, helping ensure the legal medical record contains complete, accurate data. 

 ](https://fhir.epic.com/Documentation?docId=cardiac_devices)
 

 

 [
 
## Infusion Pumps

 
Bi-directional infusion pump integration allows clinicians to send order details from Epic to auto-program the pump, and then receive documentation information from the pump, such as rate changes and volumes, to file into the patient's chart.

 ](https://fhir.epic.com/Documentation?docId=infusion_pumps_playbook)
 

 

 [
 
## International Patient Summary

 
Organizations that use Epic can generate FHIR-based IPS documents and make those documents available for patients.

 ](https://fhir.epic.com/Documentation?docId=internationalpatientsummary)
 

 

 [
 
## Medication Carousels & Robots

 
Medication Carousels & Robots are used in inpatient pharmacies as places to store medications, retrieve them, and automate dispenses to a pharmacist. 

 ](https://fhir.epic.com/Documentation?docId=medicationcarouselplaybook)
 

 

 [
 
## Medication Ordering Decision Support

 
A decision support system used in Hyperspace with CDS Hooks to pull medication orders and provide medication change recommendations.

 ](https://fhir.epic.com/Documentation?docId=medicationorderingdecisionsupport_playbook)
 

 

 [
 
## Patient Rostering

 
Strategies to build and maintain patient rosters from Epic, including event-based interfaces, Bulk FHIR, report extracts, and the FHIR List.Search API.

 ](https://fhir.epic.com/Documentation?docId=patient_roster_playbook)
 

 

 [
 
## Patient, Payer, and Provider Access

 
Patient, Provider, and Payer-to-Payer APIs enable secure access to claims and clinical data.

 ](https://fhir.epic.com/Documentation?docId=cms_access_playbook)
 

 

 [
 
## Patient-Facing / Consumer Health Apps

 
Patient-authorized health apps allow people to access, track, and share their medical records and personal health information, connecting with their MyChart credentials to healthcare provider data through TEFCA IAS or Epic on FHIR.

 ](https://fhir.epic.com/Documentation?docId=patient_facing_apps)
 

 

 [
 
## Payer Connections to Epic Providers

 
Payer connections with Epic provider systems, covering roster management, FHIR/HL7 clinical data exchange and encounter notifications, care gaps via CDS Hooks, prior authorization, and related recommended practices.

 ](https://fhir.epic.com/Documentation?docId=payerintegration)
 

 

 [
 
## Payment Device Integrations

 
The Payment Device Integration supports credit card and check scanning devices to enable real-time payment collection, with payments being processed outside of Epic by the payment processor.

 ](https://fhir.epic.com/Documentation?docId=paymentdeviceintegrations_playbook)
 

 

 [
 
## Perioperative Workflows

 
Exchange data for perioperative workflows, including surgical consults, case requests, scheduled surgeries, day-of-surgery clinical documentation, and post-operative outcomes.

 ](https://fhir.epic.com/Documentation?docId=periop_playbook)
 

 

 [
 
## Point-of-care Middleware Integrations

 
Point-of-care Middleware Integrations connect lab hardware to Epic to facilitate receiving results from point-of-care devices into Epic by translating data received by many different types of lab devices into a standard format to be consumed by Epic. 

 ](https://fhir.epic.com/Documentation?docId=pocmiddleware_playbook)
 

 

 [
 
## Printing Output Management Integrations

 
Output Management applications integrate with Epic Print Services (EPS) to collect the jobs exiting from the service that are bound for physical printer devices, which would normally be done through Windows print queues.

 ](https://fhir.epic.com/Documentation?docId=oms_playbook)
 

 

 [
 
## Prior Authorization

 
CMS-aligned CRD, DTR, and PAS APIs enable real-time coverage requirements discovery, communication of documentation templates & rules, and completion of electronic prior authorization submission to reduce manual effort and turnaround time.

 ](https://fhir.epic.com/Documentation?docId=cms_prior_auth_playbook)
 

 

 [
 
## Radiation Therapy Treatment Management System

 
Radiation Therapy Treatment Management Systems use patient imaging to create radiation therapy plans and handle radiation therapy delivery, sending treatment information to Epic.

 ](https://fhir.epic.com/Documentation?docId=radiation_therapy_tms)
 

 

 [
 
## Real Time Location Systems

 
Real-Time Location System (RTLS) allows health systems to track patients, staff, and medical devices, and to automate workflows within their Epic implementation.

 ](https://fhir.epic.com/Documentation?docId=real_time_location_systems_playbook)
 

 

 [
 
## Reference Labs

 
Common labs, pathology, specialty diagnostics, and genetic testing orders placed by providers in Epic can be sent to external reference labs that process and return results to Epic.

 ](https://fhir.epic.com/Documentation?docId=reference_labs_playbook)
 

 

 [
 
## Remote Patient Monitoring with Devices

 
Patients are monitored from home and submit data to Epic flowsheets such as vitals, via direct device connection with the Bluetooth® Generic Health Sensor or software connections with APIs and interfaces.

 ](https://fhir.epic.com/Documentation?docId=rpmtutorial)
 

 

 [
 
## Specialty Registries

 
Methods to connect to specialty registries, including SQL, HL7 interfaces, and Bulk FHIR.

 ](https://fhir.epic.com/Documentation?docId=registries_playbook)
 

 

 [
 
## Standard Lab Middleware Integrations

 
Standard Lab Middleware Integrations connect lab hardware to Epic to facilitate receiving results from lab instruments into Beaker by translating data received by many different types of lab instruments into a standard format to be consumed by Epic.

 ](https://fhir.epic.com/Documentation?docId=standardlabmiddleware_playbook)
 

 

 [
 
## Surgical Supplies

 
Ensure surgical supply readiness and accurate stock management through interfaces

 ](https://fhir.epic.com/Documentation?docId=surgical_materials_playbook)
 

 

 [
 
## Telehealth Providers 

 
Telehealth providers offer the platform as well as the healthcare staff to provide video visits to patients.

 ](https://fhir.epic.com/Documentation?docId=telehealth_providers_playbook)
 

 

 [
 
## TPN Compounders

 
Whether in-house or off-site, TPN compounding involves receiving ingredient level information via an order, and compounding precise amounts of each component.

 ](https://fhir.epic.com/Documentation?docId=tpn_compounding_playbook)
 

 

 [
 
## Vaccine Credentials / SMART Health Cards

 
A vaccine credential, commonly implemented as a SMART Health Card, is a digitally signed, machine-readable record that binds limited patient demographics with their vaccination or lab test history, allowing it to be securely verified by apps and verifiers while remaining interoperable, privacy-preserving, and usable both digitally and on paper.

 ](https://fhir.epic.com/Documentation?docId=vaccinecredential)
 

 

 [
 
## Validating AI Features

 
Methods to rigorously and continuously validate AI features using customer-specific data and Epic's open-source Trust & Assurance Suite of tools to assess model performance, fairness, drift, and clinical impact.

 ](https://fhir.epic.com/Documentation?docId=validating_ai_features_playbook)
 

 

 [
 
## Vitals Gateway and Middleware Device Integration

 
Vitals Gateway and Middleware vendors receive vitals data from devices like bedside monitors or anesthesia carts and then file that data to Epic.

 ](https://fhir.epic.com/Documentation?docId=vitalsgateway_playbook)
 

 

 [
 
## Waveform Visualization

 
Waveform visualization apps display waveform data in near real time via a web and/or mobile app that launches from Epic.

 ](https://fhir.epic.com/Documentation?docId=waveformvisualization_playbook)
 

 

 [
 
## Wayfinding

 
Wayfinding applications using a patient's appointment information, such as encounter department, to help users navigate healthcare organizations.

 ](https://fhir.epic.com/Documentation?docId=wayfinding_playbook)
 

 

 [
 
## Wound Care Apps

 
Connect with Epic by searching for or creating wounds, filing wound images, and recording wound properties and assessments in flowsheet rows.

 ](https://fhir.epic.com/Documentation?docId=woundcare)
 

 

 

 
Supporting patient-directed access and sharing of their record.
